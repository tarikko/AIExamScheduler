Expected CSV Structure:
Each row represents one student-course enrollment. Required columns:

course_code — Unique course identifier (e.g., CS101)
course_name — Human-readable course name
enrollment — Total enrollment count for the course
student_id — Student identifier (e.g., STU001)
room_name — Available room name (e.g., Amphi 1)
room_capacity — Room seating capacity
timeslot — (Optional) Date string in format YYYY-MM-DD HH:MM

Analytical Notes

Heuristics & Lessons
  • Genetic Algorithm evolves populations with roulette-wheel selection + elitism.
  • CSP uses MRV + Forward Checking to prune the most conflicting exams early.
  • Both algorithms run on the Python FastAPI backend with real-time SSE progress.
  • Fitness rewards efficient room allocation and penalizes late/consecutive exams.
Comparative Evaluation
  • All strategies stop when hard constraints (room conflicts, student overlaps) are resolved.
  • Soft penalties drive quality: consecutive same-day exams and room utilization efficiency.
  • Visual grid highlights room utilization; empty cells mean unconstrained slots.
